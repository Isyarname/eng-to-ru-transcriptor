# English-to-Russian Phonetic Transcriptor 🔤

Масштабируемый гибридный движок для автоматической транскрипции английского текста кириллицей. Проект использует кастомный декларативный DSL для описания правил транслитерации, автоматическую компиляцию в регулярные выражения с кэшированием по требованию и двухуровневую систему поиска.


## ✨ Ключевые возможности

* **Контекстно-зависимый DSL транслитерации**: Логика замены символов полностью изолирована в виде декларативных конфигураций в файле `data/transliteration_rules.py`. Правила описываются синтаксисом `Исходная_строка -> Результат`, что позволяет гибко задавать сложные контекстные переходы (например, йотирование гласных или расстановку ударений).
* **Поддержка макросов (текстовых шаблонов)**: Группы символов (согласные `C`, гласные `V`, йотированные `I`, `J`) вынесены в обычный Python-словарь в виде сырых пар строк для замены. Это позволяет лаконично описывать классы звуков в контекстных правилах (например, для расстановки ударений).
* **Автоматическая компиляция и кэширование**: Скрипт парсит текстовую строку правил, автоматически дублирует их для заглавных букв (учет регистра), компилирует в пары регулярных выражений и сериализует в файл `compiled_rules.json` при первом запуске (Lazy Initialization). При повторных вызовах готовый кэш считывается с диска с нулевой задержкой.
* **Двухуровневый гибридный поиск с нейросетевым фолбэком**: Модуль `eng_to_ipa_hybrid.py` реализует отказоустойчивую стратегию поиска с фолбэком (Fallback strategy). 
  * **Первый уровень**: Мгновенный поиск точных исключений по хэш-таблице локального словаря `en_dict.txt`.
  * **Второй уровень (Нейросетевой)**: Если слова нет в словаре, вместо устаревших жестких подстановок программа использует современный фонемный движок `gruut`. Благодаря предобученной нейросетевой G2P-модели (Grapheme-to-Phoneme) система с высокой точностью генерирует IPA-транскрипцию для редких, заимствованных или вымышленных слов на основе контекста букв. Чтобы избежать просадок по скорости, неизвестные слова объединяются в пакеты (Bulk Processing) и обрабатываются за один сеанс.
* **Контекстный преобразователь строк**: Вычислительное ядро последовательно применяет скомпилированные правила за один проход. Движок разделяет правила на простые замены и регулярные выражения, используя для простых строк высокопроизводительный нативный метод `str.replace`.
* **Фонетико-графическая адаптация**: Система не просто копирует знаки МФА, а адаптирует их под естественные правила чтения русского языка. Контекстные правила автоматически схлопывают избыточные фонетические сочетания (например, переводят `йэ` в `е`, `ʲа` в `я`, `ʲу` в `ю`), делая итоговый кириллический текст легко читаемым.

### 📐 Пайплайн обработки данных

Весь процесс работы программы делится на два независимых этапа: подготовку правил и обработку текста.

#### 1. Подготовка и кэширование правил
* **Проверка кэша**: Точка входа или внешний API-вызов обращаются к функции `convert()`, которая проверяет наличие файла `compiled_rules.json`.
* **Компиляция (`src/compiler.py`)**: Если файла нет, внутренний модуль считывает декларативные правила из `data/ipa_to_ru_rules.py`. Компилятор разворачивает лингвистические сокращения, автоматически генерирует правила для заглавных букв (учет регистра) и собирает регулярные выражения.
* **Сохранение**: Скомпилированные правила сериализуются в `compiled_rules.json`. При последующих запусках этот этап пропускается, обеспечивая старт с нулевой задержкой.

#### 2. Транскрипция текста
* **Интерфейс ввода-вывода (`main.py`)**: Точка входа для консольного запуска. Занимается исключительно чтением исходного `text.txt`, передачей строки в движок и записью результата в `output_text.txt`.
* **Оркестрация (`src/eng_to_ru.py`)**: Универсальная функция `convert()`. Если аргументы не переданы, она лениво парсит локальный словарь `data/en_US.txt` и подтягивает JSON-кэш правил, после чего координирует чистые вычисления.
* **Фонетический анализ (`src/eng_to_ipa_hybrid.py`)**: Реализует двухуровневый поиск. Если слово есть в словаре исключений — транскрипция берется оттуда, если нет — она генерируется «на лету» через библиотеку `eng_to_ipa`.
* **Транслитерация (`src/ipa_to_ru.py` + `src/transducer.py`)**: Полученный IPA-текст передается в модуль транслитерации. Чистый движок последовательно применяет скомпилированные регулярные выражения к строке, заменяя фонетические знаки на кириллицу.

## 📂 Структура проекта
```text
eng-to-ru-transcriptor/
│
├── main.py                  # Главная точка входа для запуска транскриптора
├── compiled_rules.json      # Автоматически генерируемый JSON-кэш скомпилированных правил
├── output_text.txt          # Файл с результатом работы программы (выходной текст)
├── requirements.txt
├── .gitignore
│
├── data/                    # Данные, словари и правила
│   ├── text.txt             # Исходный английский текст для транскрипции
│   ├── en_dict.txt          # Локальный словарь для быстрого поиска и исключений
│   └── transliteration_rules.py   # Описание правил транслитерации на кастомном DSL
│
└── src/                     # Исходный код модулей движка
    ├── compiler.py          # Компилятор: парсит DSL-правила и собирает пары Regex
    ├── transducer.py        # Движок трансдьюсера: последовательное применение регулярных выражений
    ├── ipa_to_ru.py         # Модуль транслитерации IPA-символов в кириллицу
    ├── eng_to_ipa_hybrid.py # Гибридный модуль: интеграция с gruut и словарем en_US.txt
    └── eng_to_ru.py         # Главный оркестратор: содержит универсальную функцию convert()
```
## 🚀 Установка и запуск

1. Клонируйте репозиторий:
```bash
git clone https://github.com
cd eng-to-ru-transcriptor
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Поместите исходный английский текст в файл `data/text.txt`.

4. Запустите транскриптор:
```bash
python main.py
```

Результат работы будет автоматически сохранен в файл `output_text.txt` в корне проекта.
